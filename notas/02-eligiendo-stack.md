# Eligiendo el Stack: GitHub como Base de Datos

Para un reto de 30 minutos, la velocidad es todo. No hay tiempo para configurar servidores o bases de datos complejas.

Mi "stack" es simple:
- **Servidor y Base de Datos:** Un repositorio público de GitHub.
- **Formato de Contenido:** Archivos de texto plano en formato Markdown.

Esto cumple con la estética "vintage hacker" y es increíblemente rápido de implementar. Cada archivo en GitHub tiene una URL "raw" única, que funciona perfectamente como un endpoint de API.
